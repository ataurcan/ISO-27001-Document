**Control**
    Information security should be integrated into project management.

![](../../PNGs/Pasted%20image%2020260201104552.png)

**Purpose** 
    To ensure information security risks related to projects and deliverables are effectively addressed in project management throughout the project life cycle.


**Guidance (Summary)** 
- Assess and treat security risks early and throughout the project. 
- Define roles and responsibilities for security within projects. 
- Include security requirements for deliverables and processes

**Other Information** 
    Aligns with ISO 21500/21502 project management practices.


**Difficulties in Control Interpretation and Application** 
- Security is added late instead of integrated from the start. 
- Project managers often lack security training. 
- Application across diverse project types is inconsistent.
