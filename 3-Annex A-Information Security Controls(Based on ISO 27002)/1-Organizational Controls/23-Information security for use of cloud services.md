
**Control** 
    Processes for acquisition, use, management and exit from cloud services should be established in accordance with the organization's information security requirements.

![](../../PNGs/Pasted%20image%2020260201150035.png)

**Purpose** 
    To specify and manage information security for the use of cloud services.

**Guidance (Summary)** 
- Define clear security responsibilities for cloud providers and the organization. 
- Review cloud provider security capabilities and certifications. 
- Monitor compliance and manage risks continuously.

**Other Information** 
    Use cloud-specific controls and comply with relevant laws and standards.

**Difficulties in Control Interpretation and Application** 
- Responsibility for security controls between provider and organization is often unclear.
- Reviewing provider certifications and SLAs is skipped or done superficially. 
- Continuous monitoring and risk reassessment for cloud services is rarely done.
