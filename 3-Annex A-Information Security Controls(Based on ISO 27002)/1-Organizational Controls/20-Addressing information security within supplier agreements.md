
**Control** 
    Relevant information security requirements should be established and agreed with each supplier based on the type of supplier relationship.

![](../../PNGs/Pasted%20image%2020260201123532.png)


**Purpose** 
    To maintain an agreed level of information security in supplier relationships.

**Guidance (Summary)** 
- Define clear security controls in contracts. 
- Cover access rights, incident reporting and compliance. 
- Review agreements regularly for adequacy.

**Other Information** 
    Critical for managing third-party risks.

**Difficulties in Control Interpretation and Application** 
- Many contracts lack detailed, enforceable security requirements. 
- Incident reporting, auditing rights, and SLA for security are often missing. 
- Agreements are not reviewed regularly or aligned with evolving threats.
