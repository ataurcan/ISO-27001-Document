
**Control** 
    Changes to information processing facilities and information systems should be subject to change management procedures.

![](../../PNGs/Pasted%20image%2020260203181934.png)


**Purpose** 
    To preserve information security when executing changes.

**Guidance (Summary)** 
- Use formal change request and approval processes. 
- Record and communicate changes to stakeholders. 
- Review changes for security impact.

**Other Information**
    Supports traceability and accountability.

**Difficulties in Control Interpretation and Application** 
- Formal change processes are bypassed for urgent fixes or updates. 
- Documentation of changes is often incomplete, reducing traceability.
- Security impact assessments are not consistently performed.
