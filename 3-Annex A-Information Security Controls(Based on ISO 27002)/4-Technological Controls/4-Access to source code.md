
**Control** 
    Read and write access to source code, development tools and software libraries should be appropriately managed.

![](../../PNGs/Pasted%20image%2020260203152948.png)


**Purpose** 
    To prevent the introduction of unauthorized functionality, avoid unintentional or malicious changes and to maintain the confidentiality of valuable intellectual property.

**Guidance (Summary)**
- Use version control systems with access restrictions.
- Monitor source code repositories. 
- Log and review changes.

**Other Information** 
    Applies to in-house and third-party developed code.

**Difficulties in Control Interpretation and Application** 
- Source code repositories lack proper access controls or monitoring.
- Developers sometimes use personal accounts or external storage. 
- Third-party access is granted without proper contractual controls.
