
**Control** 
    Principles for engineering secure systems should be established, documented, maintained and applied to any information system development activities.

![](../../PNGs/Pasted%20image%2020260203180609.png)


**Guidance (Summary)**
- Include security in functional requirements.
- Align with compliance and industry standards. 
- Document requirements and test for compliance.

**Other Information** 
    Review requirements regularly.

**Difficulties in Control Interpretation and Application**
- Security requirements are often added late or omitted in project planning. 
- Functional and security requirements are not well aligned. 
- Compliance with documented security requirements is rarely validated.
